Метод main и его дескрипторы

Всякая программа на Java, оформленная как приложение (application), должна содержать метод с именем main. Он может быть один на все приложение, а может находиться в каждом классе. Подобное размещение точки входа в каждом классе обычно применяется на сыром проекте, и при окончательной сборке эти тестовые методы main удаляются.

Дескрипторы / спецификаторы доступа

Спецификаторы доступа к методу это специальные ключевые слова языка Java, которые управляют доступом к методу

Метод main() объявляется в классе, как обычный метод, может содержать любые описания и действия, но он обязательно должен быть открытым (public), статическим (static), не иметь возвращаемого значения (void). Его аргументом обязательно должен быть массив строк (String [ ]). По традиции этот массив называют args (аргументы), хотя имя может быть любым. 


class MainClass {
        public static void main(String [ ] args) {
            .......
   }
}

Эти особенности возникают из-за того, что метод с именем main() вызывается автоматически исполняющей системой Java в самом начале выполнения приложения. А также статические элементы (static) принадлежат именно классам (коду в файле), а не объектам (сущностям в оперативной памяти). И так, как в самом начале ещё нет никаких объектов, то точка входа обязана быть статичной.

При вызове интерпретатора Java нужно указываеть класс, где объявлен метод main(), с вызова которого виртуальная машина Java начанёт выполнение.

Поскольку классов с методом main() может быть несколько, можно построить приложение с дополнительными точками входа, начиная выполнение приложения в разных ситуациях из различных классов.

Часто метод main() заносят в каждый класс с целью отладки. В этом случае в метод main() включают тесты для проверки работы всех методов класса.
